Chapter 2 Lab: Basic UI Elements and Layouts

Introduction

You will build a campus event card using stacks, spacing tools, and reusable view components.

Think of layout like arranging furniture in a room. Stacks place items in rows or columns. Spacers push items apart. Reusable views are like LEGO bricks you build once and use again.

You can complete this lab in about 15 minutes.

Learning Objectives

After completing this lab, you will be able to:

  • Refactor a large view into smaller, reusable subviews
  • Arrange UI elements with VStack, HStack, and ZStack
  • Control spacing with padding, Spacer, and Divider
  • Set size and position with frame and alignment
  • Read screen dimensions with GeometryReader
  • Explain when to use each stack type in a layout

Key Ideas

Reusable views are LEGO bricks. Build a header once, then snap it into many screens.

Stacks organize your layout. VStack stacks items like books on a shelf. HStack lines them up like photos on a wall. ZStack layers them like a sandwich.

layout concepts and simple analogies
Concept What it does Analogy
Reusable subview A small view extracted into its own struct A LEGO brick used in many builds
VStack Arranges views vertically, top to bottom Books stacked on a shelf
HStack Arranges views horizontally, left to right Photos in a row on a wall
ZStack Layers views on top of each other Layers in a sandwich
Spacer Pushes views apart to fill extra space A spring that stretches between items
Divider Draws a line between sections A ruler line between notebook sections
padding Adds space around a view The mat around a picture in a frame
frame Sets width, height, and alignment A fixed-size picture frame
GeometryReader Reads the size of the available space A measuring tape for the screen

Part 2: Extract a Reusable Header View

Topics: Reusing views code and More about Text.

You to break big views into smaller pieces. Create a separate struct called EventHeader. Move the title and subtitle into it.

import SwiftUI

struct EventHeader: View {
    let title: String
    let subtitle: String

    var body: some View {
        VStack(alignment: .leading, spacing: 4) {
            Text(title)
                .font(.largeTitle)
                .fontWeight(.heavy)
                .foregroundColor(.blue)
            Text(subtitle)
                .font(.headline)
                .fontWeight(.bold)
                .foregroundColor(.secondary)
        }
    }
}

Answer this question: Why is it helpful to extract EventHeader into its own struct? Write one or two sentences in your submission.

Part 3: Build the Event Card with Stacks

Topics: Managing space in SwiftUI, padding, frame, and Digging deeper into stacks.

Build your main card in ContentView. Use a VStack for the card body. Use an HStack for the date and location row.

struct ContentView: View {
    var body: some View {
        VStack(alignment: .leading, spacing: 16) {
            EventHeader(
                title: "SwiftUI Study Night",
                subtitle: "MCC Student Center"
            )

            Divider()

            HStack {
                Image(systemName: "calendar")
                Text("Friday, 6:00 PM")
                Spacer()
                Image(systemName: "mappin.and.ellipse")
                Text("Room 204")
            }
            .font(.subheadline)

            Text("Join us for a hands-on SwiftUI review session before the midterm.")
                .font(.body)
                .foregroundColor(.primary)
        }
        .padding()
        .frame(maxWidth: .infinity, alignment: .leading)
        .background(Color(.systemGray6))
        .cornerRadius(12)
    }
}

#Preview {
    ContentView()
}

Build and preview your card. Confirm the title, divider, and detail row all appear correctly.

Part 4: Use Spacer and Divider for Layout Control

Topics: The Spacer and The Divider.

Add a Spacer inside your HStack to push the date and location to opposite sides. Add a second Divider below the description text.

HStack {
    Label("Friday, 6:00 PM", systemImage: "calendar")
    Spacer()
    Label("Room 204", systemImage: "mappin.and.ellipse")
}
.font(.subheadline)

Divider()

Text("Free pizza for the first 20 students.")
    .font(.caption)
    .foregroundColor(.secondary)

Notice how Spacer eats up empty space between items. The Divider creates a visual break between sections.

Part 5: Layer Content with ZStack

Topics: ZStack and Overview of stacks.

Wrap your card in a ZStack to add a featured badge in the top-right corner. The badge sits on top of the card like the top slice of bread on a sandwich.

struct ContentView: View {
    var body: some View {
        ZStack(alignment: .topTrailing) {
            // Card background layer
            VStack(alignment: .leading, spacing: 16) {
                EventHeader(
                    title: "SwiftUI Study Night",
                    subtitle: "MCC Student Center"
                )
                Divider()
                HStack {
                    Label("Friday, 6:00 PM", systemImage: "calendar")
                    Spacer()
                    Label("Room 204", systemImage: "mappin.and.ellipse")
                }
                .font(.subheadline)
                Divider()
                Text("Join us for a hands-on SwiftUI review session.")
                    .font(.body)
            }
            .padding()
            .frame(maxWidth: .infinity, alignment: .leading)
            .background(Color(.systemGray6))
            .cornerRadius(12)

            // Badge layer on top
            Text("Featured")
                .font(.caption)
                .fontWeight(.bold)
                .foregroundColor(.white)
                .padding(.horizontal, 8)
                .padding(.vertical, 4)
                .background(Color.orange)
                .cornerRadius(6)
                .padding(12)
        }
        .padding()
    }
}

Part 6: Explore Frame, Alignment, and GeometryReader

Topics: The .frame modifier, The .alignment modifier, and Using GeometryReader.

Task 6.1: Alignment with frame

Add a row that shows how alignment changes text position inside a fixed frame. Use three boxes with leading, center, and trailing alignment.

struct AlignmentDemo: View {
    var body: some View {
        VStack(spacing: 12) {
            Text("Leading")
                .frame(width: 280, height: 40, alignment: .leading)
                .background(Color.blue.opacity(0.3))
            Text("Center")
                .frame(width: 280, height: 40, alignment: .center)
                .background(Color.green.opacity(0.3))
            Text("Trailing")
                .frame(width: 280, height: 40, alignment: .trailing)
                .background(Color.orange.opacity(0.3))
        }
    }
}

Add AlignmentDemo() below your event card in ContentView for preview testing. Remove it before final submission if you prefer a cleaner card.

Task 6.2: GeometryReader stretch task

Use GeometryReader to display the width of the card. This is like using a measuring tape on the screen.

GeometryReader { geometry in
    Text("Card width: \(Int(geometry.size.width)) points")
        .font(.caption)
        .foregroundColor(.secondary)
        .frame(maxWidth: .infinity, alignment: .center)
}

Place this below your card. Resize the preview to different device sizes. Note how the width value changes.

Troubleshooting

Views Overlap in ZStack

Check your ZStack alignment parameter. Use .topTrailing for a top-right badge. Add .padding() to the badge so it does not touch the screen edge.

Spacer Has No Effect

A Spacer only works inside a stack with extra room. Make sure the parent HStack or VStack is wide or tall enough to expand.

GeometryReader Collapses

GeometryReader expands to fill all available space. Wrap it in a frame(height:) if it pushes other views off screen. Try .frame(height: 30) on the reader.

Card Looks Cramped on Small Screens

Add outer .padding() to ContentView. Use .frame(maxWidth: .infinity) so the card stretches to the screen width.
