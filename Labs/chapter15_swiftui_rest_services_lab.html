Chapter 15 Lab: Consuming REST Services in SwiftUI

Introduction

Many apps load data from the internet. A weather app fetches forecasts. A campus app might load event posts from a server. This lab teaches that flow in SwiftUI.

You will build CampusFeed, an app that downloads announcement-style posts from a public practice API and shows them in a list. The text is sample data, but the networking steps are the same ones used in production apps.

Learning Objectives

After completing this lab, you will be able to:

  • Explain REST and HTTP GET requests in plain language
  • Map JSON to Swift types with Decodable and Codable
  • Download data with URLSession and async/await
  • Set request headers such as Accept
  • Handle network and server errors without crashing
  • Update the UI with ObservableObject and @Published
  • Describe why HTTPS helps protect users on public networks

Key Ideas

REST is like a campus bulletin API. You ask for posts with a standard URL. The server sends back JSON text your app can read.

HTTP GET is a read-only request. You are saying, "Please send me the list." You are not changing anything on the server.

URLSession is the campus mail truck. It drives to the server address, picks up the response, and brings it back to your app.

JSON is the envelope format. Decodable is the clerk who opens the envelope and fills out your Swift forms.

HTTPS is the locked truck. It helps stop strangers on the same Wi‑Fi from reading or changing messages in transit.

REST networking concepts and simple analogies
Concept What it does Analogy
REST A style of web APIs that use URLs and HTTP verbs A bulletin board with a standard address for each list
HTTP GET Requests data without changing server state Asking the front desk for today's flyers
URL The web address your request visits The building address on an envelope
Port A numbered door on a server (443 for HTTPS) Which campus entrance the truck uses
JSON Text format for structured data A typed form inside the envelope
Decodable Turns JSON into Swift structs or classes A clerk copying form fields into your database
URLSession Apple's tool for HTTP network calls The mail truck that fetches responses
HTTP header Extra instructions on a request or response A sticky note that says "I expect JSON back"
@Published Marks ViewModel properties that refresh the UI A bell that rings when new posts arrive
URLProtocol Hooks network calls for testing or mocking A practice mail route before the real truck leaves

Part 2: Create a Decodable Model

Topics: Codable, Encodable, and Decodable protocols.

Add CampusAnnouncement.swift. The API returns objects with id, title, and body. Match those names in Swift.

import Foundation

struct CampusAnnouncement: Identifiable, Decodable {
    let id: Int
    let title: String
    let body: String
}

Decodable means Swift can build this struct from JSON automatically when property names match. You only need custom CodingKeys when JSON names differ from Swift names.

Part 3: Build a Network Service with URLSession

Topics: Using URLSession; implementing HTTP methods; setting headers.

Create AnnouncementService.swift. Keep network code out of the view, just like Chapter 10 taught.

import Foundation

enum AnnouncementError: Error {
    case badServerResponse
    case decodingFailed
}

protocol AnnouncementServiceProtocol {
    func fetchAnnouncements() async throws -> [CampusAnnouncement]
}

struct LiveAnnouncementService: AnnouncementServiceProtocol {
    private let url = URL(string: "https://jsonplaceholder.typicode.com/posts")!

    func fetchAnnouncements() async throws -> [CampusAnnouncement] {
        var request = URLRequest(url: url)
        request.httpMethod = "GET"
        request.setValue("application/json", forHTTPHeaderField: "Accept")

        let (data, response) = try await URLSession.shared.data(for: request)

        guard let httpResponse = response as? HTTPURLResponse,
              httpResponse.statusCode == 200 else {
            throw AnnouncementError.badServerResponse
        }

        do {
            return try JSONDecoder().decode([CampusAnnouncement].self, from: data)
        } catch {
            throw AnnouncementError.decodingFailed
        }
    }
}

The Accept header tells the server you expect JSON back. URLSession.shared.data(for:) is the modern async way to perform the GET request.

Add a mock service for previews and offline testing:

struct MockAnnouncementService: AnnouncementServiceProtocol {
    func fetchAnnouncements() async throws -> [CampusAnnouncement] {
        [

}

Part 4: Add a ViewModel with @Published State

Topics: Watching UI changes with ObservableObject and @Published.

Create AnnouncementListViewModel.swift. The ViewModel owns loading state and calls the service.

import Foundation

@MainActor
final class AnnouncementListViewModel: ObservableObject {
    @Published var announcements: [CampusAnnouncement] = []
    @Published var isLoading = false
    @Published var errorMessage: String?

    private let service: AnnouncementServiceProtocol

    init(service: AnnouncementServiceProtocol = LiveAnnouncementService()) {
        self.service = service
    }

    func loadAnnouncements() async {
        isLoading = true
        errorMessage = nil

        do {
            announcements = try await service.fetchAnnouncements()
        } catch {
            errorMessage = "Could not load announcements. Check your connection and try again."
        }

        isLoading = false
    }
}

@MainActor keeps UI updates on the main thread. @Published tells SwiftUI to redraw when the list or error message changes.

Part 5: Build the SwiftUI Feed Screen

Topics: Integrating REST services into SwiftUI views.

Create AnnouncementListView.swift. Keep the screen logic separate from the app entry point so previews are easy.

import SwiftUI

struct AnnouncementListView: View {
    @ObservedObject var viewModel: AnnouncementListViewModel

    var body: some View {
        NavigationStack {
            Group {
                if viewModel.isLoading && viewModel.announcements.isEmpty {
                    ProgressView("Loading announcements…")
                } else if let errorMessage = viewModel.errorMessage,
                          viewModel.announcements.isEmpty {
                    ContentUnavailableView(
                        "Unable to Load",
                        systemImage: "wifi.exclamationmark",
                        description: Text(errorMessage)
                    )
                } else {
                    List(viewModel.announcements) { announcement in
                        VStack(alignment: .leading, spacing: 4) {
                            Text(announcement.title)
                                .font(.headline)
                                .lineLimit(2)
                            Text(announcement.body)
                                .font(.subheadline)
                                .foregroundStyle(.secondary)
                                .lineLimit(2)
                        }
                        .padding(.vertical, 4)
                    }
                    .refreshable {
                        await viewModel.loadAnnouncements()
                    }
                }
            }
            .navigationTitle("Campus Feed")
            .toolbar {
                ToolbarItem(placement: .navigationBarTrailing) {
                    Button("Refresh") {
                        Task {
                            await viewModel.loadAnnouncements()
                        }
                    }
                }
            }
            .task {
                await viewModel.loadAnnouncements()
            }
        }
    }
}

struct ContentView: View {
    @StateObject private var viewModel = AnnouncementListViewModel()

    var body: some View {
        AnnouncementListView(viewModel: viewModel)
    }
}

#Preview("Mock Data") {
    let viewModel = AnnouncementListViewModel(service: MockAnnouncementService())
    return AnnouncementListView(viewModel: viewModel)
        .task { await viewModel.loadAnnouncements() }
}

Troubleshooting

The List Is Empty With No Error

Check that .task or Refresh calls loadAnnouncements(). Confirm the URL uses https:// and returns status 200. Print data.count in the service to verify bytes arrived.

Decoding Failed Error

Compare your struct property names to the JSON keys from the API. The sample API uses id, title, and body. Extra JSON fields are fine. Missing or mismatched names cause decode failures.

App Transport Security Blocked Load

Do not use plain http:// URLs in production apps. Stick with https:// endpoints like JSONPlaceholder. Avoid disabling ATS unless a course instructor explicitly allows it for a local test server.

UI Does Not Update After Load

Mark the ViewModel with @MainActor. Use @StateObject in the root view and @ObservedObject in child views. Properties that drive the UI must have @Published.

Preview Shows Empty List

Inject MockAnnouncementService() into the ViewModel for previews. Call await viewModel.loadAnnouncements() inside .task on the preview.
