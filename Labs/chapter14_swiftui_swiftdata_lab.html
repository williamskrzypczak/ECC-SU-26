Chapter 14 Lab: An Introduction to SwiftData

Introduction

You will build CampusNotes, a simple app that saves notes on the device. Notes stay after you close the app. That is persistence — data that outlives one launch.

SwiftData is Apple's modern way to store app data with SwiftUI. You describe your data with Swift classes instead of drawing diagrams in a separate editor.

Learning Objectives

After completing this lab, you will be able to:

  • Explain how SwiftData differs from Core Data in plain language
  • Define a persistent model with the @Model macro
  • Attach a ModelContainer to your app entry point
  • Fetch saved data with @Query in a SwiftUI view
  • Insert and delete records using modelContext
  • Edit saved objects with @Bindable two-way bindings
  • Describe one-to-many relationships and cascade delete rules

Key Ideas

SwiftData is like a smart campus locker. You label what goes inside with Swift code. The system handles the lock, shelf, and inventory list.

Core Data is the older manual ledger system. It still works, but setup takes more steps and separate tools.

ModelContainer is the storage building. ModelContext is the clerk at the front desk who files, updates, and removes items.

@Query is a live bulletin board. When data changes in storage, the board updates on its own.

SwiftData concepts and simple analogies
Concept What it does Analogy
@Model Marks a Swift class as data SwiftData can save A locker label that tells the system what belongs inside
ModelContainer Owns the database file and schema for your models The storage building on campus
ModelContext Performs insert, update, and delete operations The clerk who files papers at the front desk
@Query Loads data into a view and refreshes when data changes A bulletin board that updates when new flyers arrive
@Bindable Creates two-way bindings to a stored model in a form An editable sticky note still pinned to the board
@Relationship Links two models, such as a folder and its notes A binder that holds many sheets of paper
Cascade delete Deleting a parent can delete its children automatically Throwing away a binder also tosses the pages inside
#Predicate Filters which records @Query loads Asking the clerk for only pinned notes

Part 2: Create a @Model Class

Topics: Creating data models with @Model.

Add a new Swift file named CampusNote.swift. Import SwiftData and mark the class with @Model.

import Foundation
import SwiftData

@Model
final class CampusNote {
    var title: String
    var body: String
    var createdAt: Date
    var isPinned: Bool

    init(title: String, body: String = "", isPinned: Bool = false) {
        self.title = title
        self.body = body
        self.createdAt = Date()
        self.isPinned = isPinned
    }
}

The @Model macro turns this class into something SwiftData can store on disk. You write normal Swift properties. SwiftData handles the SQLite details behind the scenes.

Part 3: Wire Up the ModelContainer

Topics: SwiftData and SwiftUI integration; fetching data.

Open your app file (for example, CampusNotesApp.swift). Import SwiftData and attach .modelContainer(for:) to the WindowGroup.

import SwiftUI
import SwiftData

@main
struct CampusNotesApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        .modelContainer(for: CampusNote.self)
    }
}

This one line creates storage for CampusNote and injects a ModelContext into the SwiftUI environment. Every child view can reach the clerk at the desk through @Environment(\.modelContext).

Part 4: Display Notes with @Query

Topics: Fetching data; filtering with predicates.

Replace the default ContentView with a list that reads saved notes. Use @Query to fetch them sorted by date.

import SwiftUI
import SwiftData

struct ContentView: View {
    @Environment(\.modelContext) private var modelContext
    @Query(sort: \CampusNote.createdAt, order: .reverse) private var notes: [CampusNote]

    var body: some View {
        NavigationStack {
            List(notes) { note in
                VStack(alignment: .leading, spacing: 4) {
                    Text(note.title)
                        .font(.headline)
                    Text(note.createdAt.formatted(date: .abbreviated, time: .shortened))
                        .font(.caption)
                        .foregroundStyle(.secondary)
                }
            }
            .navigationTitle("Campus Notes")
            .overlay {
                if notes.isEmpty {
                    ContentUnavailableView(
                        "No Notes Yet",
                        systemImage: "note.text",
                        description: Text("Add your first campus note.")
                    )
                }
            }
        }
    }
}

Run the app. The list is empty because nothing is saved yet. That is expected.

Optional stretch: Add a second @Query that only loads pinned notes using a predicate:

@Query(filter: #Predicate<CampusNote> { $0.isPinned == true },
      sort: \CampusNote.createdAt, order: .reverse)
private var pinnedNotes: [CampusNote]

Part 5: Add and Delete Notes

Topics: ModelContext insert and delete operations.

Add a text field and buttons so users can create notes. Use modelContext.insert(_:) to save and swipe-to-delete to remove.

struct ContentView: View {
    @Environment(\.modelContext) private var modelContext
    @Query(sort: \CampusNote.createdAt, order: .reverse) private var notes: [CampusNote]
    @State private var newTitle = ""

    var body: some View {
        NavigationStack {
            List {
                ForEach(notes) { note in
                    NavigationLink(value: note) {
                        VStack(alignment: .leading, spacing: 4) {
                            Text(note.title)
                                .font(.headline)
                            if note.isPinned {
                                Text("Pinned")
                                    .font(.caption)
                                    .foregroundStyle(.orange)
                            }
                        }
                    }
                }
                .onDelete(perform: deleteNotes)
            }
            .navigationDestination(for: CampusNote.self) { note in
                NoteDetailView(note: note)
            }
            .navigationTitle("Campus Notes")
            .toolbar {
                ToolbarItem(placement: .navigationBarTrailing) {
                    Button("Sample") {
                        addSampleNote()
                    }
                }
            }
            .safeAreaInset(edge: .bottom) {
                HStack {
                    TextField("Note title", text: $newTitle)
                        .textFieldStyle(.roundedBorder)
                    Button("Add") {
                        addNote()
                    }
                    .disabled(newTitle.trimmingCharacters(in: .whitespaces).isEmpty)
                }
                .padding()
            }
        }
    }

    private func addNote() {
        let trimmed = newTitle.trimmingCharacters(in: .whitespaces)
        guard !trimmed.isEmpty else { return }
        modelContext.insert(CampusNote(title: trimmed))
        newTitle = ""
    }

    private func addSampleNote() {
        modelContext.insert(
            CampusNote(title: "Study for MAD 157", body: "Review SwiftData lab.")
        )
    }

    private func deleteNotes(at offsets: IndexSet) {
        for index in offsets {
            modelContext.delete(notes[index])
        }
    }
}

Add a note, then force quit the simulator and relaunch. Your note should still appear. That proves persistence works.

Part 6: Edit a Note with @Bindable

Topics: Binding changes with @Bindable and @Observable models.

Create NoteDetailView.swift. Use @Bindable so text fields write changes back to the saved model.

import SwiftUI
import SwiftData

struct NoteDetailView: View {
    @Bindable var note: CampusNote

    var body: some View {
        Form {
            TextField("Title", text: $note.title)
            TextField("Body", text: $note.body, axis: .vertical)
                .lineLimit(3...8)
            Toggle("Pin this note", isOn: $note.isPinned)
        }
        .navigationTitle("Edit Note")
    }
}

Tap a note in the list, change the title, go back, and tap again. The edit should stick without extra save code. SwiftData autosaves when the context changes.

Why @Bindable instead of @State? @State is for short-lived view data. @Bindable connects form controls to a persistent object that lives in the database.

Part 7: Relationships and Cascade Delete

Topics: One-to-many relationships; cascading deletion rules.

Real apps group data. A folder holds many notes. Add a second model and link it to CampusNote.

Create CampusFolder.swift:

import Foundation
import SwiftData

@Model
final class CampusFolder {
    var name: String

    @Relationship(deleteRule: .cascade, inverse: \CampusNote.folder)
    var notes: [CampusNote]

    init(name: String, notes: [CampusNote] = []) {
        self.name = name
        self.notes = notes
    }
}

Add an optional folder property to CampusNote:

var folder: CampusFolder?

Update your app file so the container knows about both models:

.modelContainer(for: [CampusNote.self, CampusFolder.self])

In addSampleNote(), create a folder and attach a note to it:

private func addSampleNote() {
    let folder = CampusFolder(name: "MAD 157")
    let note = CampusNote(title: "Study SwiftData", body: "Finish lab by Friday.")
    note.folder = folder
    folder.notes.append(note)
    modelContext.insert(folder)
    modelContext.insert(note)
}

The .cascade rule means deleting a folder also deletes its notes — like throwing away a binder with pages still inside.

Troubleshooting

@Query Returns an Empty List After Adding Data

Confirm .modelContainer(for:) is on the WindowGroup, not on a child view alone. Check that you call modelContext.insert(_:) and that your model has @Model at the class level.

Cannot Find SwiftData Types

Add import SwiftData to every file that uses @Model, @Query, or modelContext. Set the deployment target to iOS 17 or later.

Edits in NoteDetailView Do Not Save

Use @Bindable var note, not let note without a binding wrapper. Bind text fields with $note.title and $note.body.

App Crashes After Adding CampusFolder

When you add a new model, update .modelContainer(for:) to include every @Model type. Clean the build folder and delete the app from the simulator to reset old storage during development.

Relationship Errors or Missing Inverse

Both sides of a relationship must link to each other. The inverse: parameter on @Relationship must point to the property on the other model.
