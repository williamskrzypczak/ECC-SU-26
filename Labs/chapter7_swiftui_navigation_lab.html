Chapter 7 Lab: All About Navigation

Introduction

You will build a campus building directory that lets students tap a row to drill into details, jump to a building with code, and open an About screen from the toolbar.

Navigation is how users move deeper into an app and return. Think of it as walking down hallways and opening doors, not teleporting to a new app.

You can complete this lab in about 15 minutes.

Learning Objectives

After completing this lab, you will be able to:

  • Wrap a screen in NavigationStack to enable push navigation
  • Use value-based NavigationLink to pass data to a detail screen
  • Define destinations with .navigationDestination(for:)
  • Set titles and toolbar buttons with .navigationTitle and .toolbar
  • Control navigation in code with NavigationPath
  • Explain the difference between old NavigationView and modern NavigationStack

Key Ideas

NavigationStack is like a stack of cafeteria trays. You place a new tray on top when you go forward. You remove the top tray when you tap Back.

NavigationLink is a labeled door. Tap it and you walk into the next room. With the modern pattern, the link passes a value (like a building name) instead of embedding the whole next screen inside the link.

NavigationPath is the GPS history of every hallway you walked down. Your code can append or remove steps to jump ahead or return home.

navigation concepts and simple analogies
Concept What it does Analogy
NavigationStack Manages a stack of screens with a Back button A stack of cafeteria trays — top screen is visible
NavigationLink (value) Pushes a hashable value onto the stack when tapped A door label that tells the stack which room to open
.navigationDestination Maps a value type to the view that should appear The floor plan that says "Library ID opens LibraryDetailView"
.navigationTitle Sets the title in the navigation bar The sign above each hallway
.toolbar Adds buttons to the navigation bar Extra buttons on the hallway sign (Help, Menu)
NavigationPath Stores the full navigation history for code control Your step-by-step walking directions
NavigationView (legacy) Older navigation container used before iOS 16 An older campus map that still works but lacks new features
NavigationSplitView Two- or three-column layout for iPad and Mac A bulletin board with a sidebar list and detail panel

Prerequisites

Complete Chapters 1 through 6 first. Create a new SwiftUI app named CampusNavigator. Target iOS 16 or later so you can use NavigationStack.

Lab Scenario: Campus Building Directory

You will build a simple directory of campus buildings. The home screen lists buildings in a List. Tapping a row pushes a detail screen. A toolbar button opens an About screen. A "Go to Library" button jumps there without a tap on the list.

Lab Instructions

Part 1: Create a Building Model and List

Topics: Navigation as a concept; presenting views with NavigationLink.

Start with a small data model. Each building needs a unique id, a name, and a short description.

import SwiftUI

struct CampusBuilding: Identifiable, Hashable {
    let id = UUID()
    let name: String
    let description: String
    let roomCount: Int
}

Add sample data and a root list inside a NavigationStack. Use the modern value-based link pattern.

struct ContentView: View {
    private let buildings = [
        CampusBuilding(name: "Library", description: "Study rooms and research help.", roomCount: 42),
        CampusBuilding(name: "Science Hall", description: "Labs and lecture halls.", roomCount: 28),
        CampusBuilding(name: "Student Center", description: "Dining, clubs, and events.", roomCount: 15),
        CampusBuilding(name: "Gymnasium", description: "Fitness and intramural sports.", roomCount: 8)
    ]

    var body: some View {
        NavigationStack {
            List(buildings) { building in
                NavigationLink(building.name, value: building)
            }
            .navigationTitle("Campus Buildings")
        }
    }
}

Run the app. The list appears, but tapping a row does nothing useful yet. You have not told the stack which view to show for a CampusBuilding value.

Part 2: Add a Detail View with navigationDestination

Topics: Using navigationDestination with NavigationStack.

Create a detail view that reads the selected building. Then register it with .navigationDestination(for:) on the list.

struct BuildingDetailView: View {
    let building: CampusBuilding

    var body: some View {
        VStack(alignment: .leading, spacing: 12) {
            Text(building.name)
                .font(.largeTitle)
            Text(building.description)
                .foregroundColor(.secondary)
            Text("Rooms: \(building.roomCount)")
                .font(.headline)
        }
        .padding()
        .navigationTitle(building.name)
    }
}

Attach the destination modifier inside your NavigationStack, right after the List:

List(buildings) { building in
    NavigationLink(building.name, value: building)
}
.navigationDestination(for: CampusBuilding.self) { building in
    BuildingDetailView(building: building)
}
.navigationTitle("Campus Buildings")

Now tapping a row pushes the detail screen. The Back button appears automatically. This is the pattern The recommended approach for iOS 16 and later.

Part 3: Add a Toolbar and About Screen

Topics: Showing a navigation bar title; adding buttons to the navigation bar.

Add toolbar buttons to the navigation bar. Use navigationDestination(isPresented:) to push the About screen when the user taps Help.

struct AboutView: View {
    var body: some View {
        Text("Campus Navigator helps students find buildings on campus.")
            .padding()
            .navigationTitle("About")
    }
}

struct ContentView: View {
    @State private var showingAbout = false

    // buildings array stays the same

    var body: some View {
        NavigationStack {
            List(buildings) { building in
                NavigationLink(building.name, value: building)
            }
            .navigationDestination(for: CampusBuilding.self) { building in
                BuildingDetailView(building: building)
            }
            .navigationDestination(isPresented: $showingAbout) {
                AboutView()
            }
            .navigationTitle("Campus Buildings")
            .toolbar {
                ToolbarItem(placement: .navigationBarTrailing) {
                    Button("Help") {
                        showingAbout = true
                    }
                }
            }
        }
    }
}

The toolbar button is like a shortcut sign on the hallway wall. It triggers navigation without a list row.

Part 4: Navigate Programmatically with NavigationPath

Topics: Using NavigationPath to control the navigation stack.

Sometimes your app must navigate from code — after a search, a notification, or a "featured" button. Bind a NavigationPath to the stack and append values to it.

struct ContentView: View {
    @State private var navPath = NavigationPath()
    @State private var showingAbout = false

    private let buildings = [
        CampusBuilding(name: "Library", description: "Study rooms and research help.", roomCount: 42),
        CampusBuilding(name: "Science Hall", description: "Labs and lecture halls.", roomCount: 28),
        CampusBuilding(name: "Student Center", description: "Dining, clubs, and events.", roomCount: 15),
        CampusBuilding(name: "Gymnasium", description: "Fitness and intramural sports.", roomCount: 8)
    ]

    var body: some View {
        NavigationStack(path: $navPath) {
            VStack(spacing: 16) {
                Button("Go to Library") {
                    if let library = buildings.first(where: { $0.name == "Library" }) {
                        navPath.append(library)
                    }
                }
                .buttonStyle(.borderedProminent)

                List(buildings) { building in
                    NavigationLink(building.name, value: building)
                }
            }
            .navigationDestination(for: CampusBuilding.self) { building in
                BuildingDetailView(building: building)
            }
            .navigationDestination(isPresented: $showingAbout) {
                AboutView()
            }
            .navigationTitle("Campus Buildings")
            .toolbar {
                ToolbarItem(placement: .navigationBarTrailing) {
                    Button("Help") {
                        showingAbout = true
                    }
                }
            }
        }
    }
}

Test the "Go to Library" button. It should open the same detail screen as tapping the list row. You are now controlling the tray stack from code.

Part 5: Add a "Back to Home" Button on the Detail Screen

Topics: Direct navigation stack manipulation with NavigationPath.

This lab shows how to pop multiple screens at once with navPath.removeLast(). Pass the path into the detail view so a button can jump straight home.

struct BuildingDetailView: View {
    let building: CampusBuilding
    @Binding var navPath: NavigationPath

    var body: some View {
        VStack(alignment: .leading, spacing: 12) {
            Text(building.name)
                .font(.largeTitle)
            Text(building.description)
                .foregroundColor(.secondary)
            Text("Rooms: \(building.roomCount)")
                .font(.headline)

            Button("Back to Building List") {
                navPath.removeLast()
            }
            .buttonStyle(.bordered)
            .padding(.top, 8)
        }
        .padding()
        .navigationTitle(building.name)
    }
}

Update your destination closure to pass the binding:

.navigationDestination(for: CampusBuilding.self) { building in
    BuildingDetailView(building: building, navPath: $navPath)
}

The system Back button and your custom button both remove one tray from the stack.

Part 6: Optional Stretch — NavigationSplitView on iPad

Topics: NavigationSplitView — multicolumn navigation.

On iPad, apps often show a sidebar and a detail panel side by side. This section introduces NavigationSplitView for this layout. If you have an iPad simulator, try wrapping your content like this:

NavigationSplitView {
    List(buildings) { building in
        NavigationLink(building.name, value: building)
    }
    .navigationTitle("Buildings")
} detail: {
    Text("Select a building")
        .foregroundColor(.secondary)
}

This is optional. Focus on the iPhone NavigationStack flow first.

Troubleshooting

Tapping a Row Does Nothing

Confirm you added .navigationDestination(for: CampusBuilding.self). The type in the modifier must match the type you pass in NavigationLink. Your model must conform to Hashable.

NavigationStack Is Unavailable

NavigationStack requires iOS 16 or later. Check your deployment target in Xcode under the project settings General tab.

Go to Library Button Has No Effect

Make sure you passed path: $navPath to NavigationStack. Call navPath.append(library) with a real CampusBuilding instance from your array.

About Screen Does Not Appear

Check that showingAbout is declared with @State and that you used $showingAbout in navigationDestination(isPresented:). Set it to true inside the Help button action.

Duplicate Navigation Titles

Set .navigationTitle on the root list and on each pushed view. Only the top screen's title shows in the bar.
