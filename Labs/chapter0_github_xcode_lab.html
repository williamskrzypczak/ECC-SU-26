Chapter 0 Lab: GitHub and Xcode Source Control

Introduction

You will learn how to save, share, and sync your code using GitHub and Xcode. Git keeps a history of every change you make. GitHub stores a copy online so you never lose your work.

Think of Git like the version history in Google Docs. Each save point is called a commit. GitHub is the cloud folder where your project lives. You can complete this lab in about 15 minutes. Complete it before the Chapter 1 SwiftUI lab.

You will use Xcode menus only. You do not need the Terminal for this lab.

Learning Objectives

After completing this lab, you will be able to:

  • Explain repository, commit, push, pull, branch, merge, and remote in your own words
  • Sign in to GitHub from Xcode
  • Create a local Git repository and connect it to GitHub
  • Commit and push changes from Xcode
  • Pull changes from GitHub into Xcode
  • Create a branch, make a change, and merge it back to main

Key Ideas

Git saves snapshots. Each commit is a saved version of your project with a short note. You can go back to any snapshot if something breaks.

GitHub is your cloud backup. Push sends your commits online. Pull downloads changes someone else made. Branches let you try ideas without breaking the main copy.

Git concepts and simple analogies
Concept What it does Analogy
Repository Your project folder plus its full change history A binder that stores every draft of a paper
Commit A saved snapshot with a message describing the change Clicking "Save version" with a short note
Remote A copy of your repo hosted on GitHub A cloud backup of your binder
Push Send your local commits to GitHub Upload your latest draft to the cloud
Pull Download commits from GitHub to your Mac Download a teammate's updates
Branch A separate line of work inside the same project A sticky-note draft before adding it to the final paper
Merge Combine changes from one branch into another Folding your sticky-note edits into the final draft

Prerequisites

You need a Mac with Xcode 15 or later installed. You also need a free GitHub account. Create one at github.com/join if you do not have one yet.

Lab Instructions

Part 1: Set Up GitHub and Xcode

Topics: GitHub account and connecting Xcode to GitHub.

  1. Open Xcode.
  2. Choose Xcode → Settings (or Preferences on older versions).
  3. Click the Accounts tab.
  4. Click the + button at the bottom left.
  5. Select GitHub and sign in with your GitHub username and password or token.
  6. Confirm your GitHub account appears in the accounts list.

Take a screenshot showing your GitHub account listed in Xcode Settings. You will submit this later.

Part 2: Create a Project and Connect to GitHub

Topics: Local repository, remote repository, and initial push.

  1. In Xcode, choose File → New → Project.
  2. Select the Multiplatform tab. Choose the App template. Click Next.
  3. Set the product name to GitPractice.
  4. Set Interface to SwiftUI and Language to Swift.
  5. Check Source Control: Create Git repository on my Mac.
  6. Choose a save location and click Create.
  7. Choose Source Control → New Remote Repository….
  8. Sign in if prompted. Set the repository name to GitPractice. Keep it public. Click Create.
  9. When Xcode asks to push, choose Push. If it does not ask, choose Source Control → Push.
  10. Open your browser. Go to github.com/your-username/GitPractice. Confirm your project files appear online.

If New Remote Repository is not available, create an empty repo on GitHub.com first. Then in Xcode choose Source Control → Git Practice → Configure Git Practice Remote… and paste the repo URL.

Part 3: Commit a Change

Topics: Staging changes and writing commit messages.

Open ContentView.swift. Replace the default text with your name, like this:

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Text("Git Practice App")
                .font(.title)
            Text("Built by Your Name")
                .font(.headline)
        }
        .padding()
    }
}

#Preview {
    ContentView()
}

Replace Your Name with your actual name. Then commit the change:

  1. Click the Source Control navigator icon in the left sidebar (it looks like a branch).
  2. Choose Source Control → Commit… (or press Cmd+Option+C).
  3. Review the diff. Make sure ContentView.swift is checked.
  4. Type a commit message: Add my name to welcome screen.
  5. Click Commit.

A good commit message describes what changed in plain English. Avoid messages like "update" or "fix stuff."

Part 4: Push Your Commit

Topics: Sending local commits to GitHub.

  1. Choose Source Control → Push.
  2. Wait for the push to finish. Check the status bar at the top of Xcode for confirmation.
  3. Open your repo on GitHub.com. Click Commits or open ContentView.swift and view its history.
  4. Confirm your commit message appears on GitHub.

Take a screenshot of your commit visible on GitHub.com.

Part 5: Pull a Change from GitHub

Topics: Downloading remote changes to your Mac.

Sometimes you edit files on GitHub.com instead of in Xcode. Pull brings those changes to your Mac.

  1. On GitHub.com, open your GitPractice repo.
  2. Click README.md. Click the pencil icon to edit.
  3. Add one line, such as: MAD 157 — Chapter 0 GitHub Lab.
  4. Scroll down. Type a commit message like Add course note to README. Click Commit changes.
  5. Return to Xcode. Choose Source Control → Pull.
  6. In the Project Navigator, confirm README.md now appears in your project folder.

Take a screenshot showing README.md in your Xcode project after the pull.

Part 6: Create a Branch

Topics: Working on a separate line of development.

Branches let you try new ideas without touching main. You will add a tagline on a branch called add-tagline.

  1. Choose Source Control → New Branch….
  2. Enter the branch name: add-tagline. Click Create.
  3. Confirm the branch picker in the Xcode toolbar shows add-tagline.
  4. In ContentView.swift, add a second line of text inside the VStack:
Text("Learning Git one commit at a time.")
    .font(.subheadline)
    .foregroundColor(.secondary)
  1. Commit the change with the message: Add tagline on add-tagline branch.

Take a screenshot showing you are on the add-tagline branch with the new text in your code.

Part 7: Merge Your Branch

Topics: Combining branch work back into main.

  1. Click the branch picker in the Xcode toolbar. Switch back to main.
  2. Choose Source Control → Merge….
  3. Select the add-tagline branch. Click Merge.
  4. Open ContentView.swift. Confirm both your name and the tagline appear.
  5. Choose Source Control → Push to send the merged code to GitHub.
  6. On GitHub.com, confirm main shows all your changes.

Optional stretch: On GitHub.com, delete the merged add-tagline branch to keep your repo tidy.

Troubleshooting

GitHub Account Not Showing in Xcode

Open Xcode → Settings → Accounts. Remove the account and add it again. Make sure you complete the sign-in flow in your browser.

Push Rejected or Failed

GitHub may have changes you do not have locally. Choose Source Control → Pull first, then push again. If Xcode shows a conflict, ask your instructor for help.

Wrong Branch Selected

Check the branch name in the Xcode toolbar. Switch branches using the branch picker or Source Control → Switch to Branch….

Cannot Merge — Uncommitted Changes

Commit or discard your current changes before merging. Open the Source Control navigator to see files waiting to be committed.

New Remote Repository Option Missing

Make sure you are signed in to GitHub under Settings → Accounts. Create an empty repo on GitHub.com manually, then add it as a remote in Xcode.
