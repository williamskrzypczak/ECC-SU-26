Chapter 6 Lab: Tab Bars and Modal View Presentation

Introduction

You will build a small campus app with a tab bar and modal screens that slide up, pop over, or ask for confirmation.

Most real apps have more than one main screen. Tab bars let users switch sections quickly. Modal views handle temporary tasks without leaving the app entirely.

Learning Objectives

After completing this lab, you will be able to:

  • Build a standard tab bar with TabView and tabItem
  • Present a modal sheet with .sheet(isPresented:)
  • Dismiss a sheet using @Environment(\.dismiss)
  • Show a confirmation dialog with .alert
  • Display a popover with .popover
  • Explain when to use a sheet, alert, or popover

Key Ideas

TabView is like labeled folders in a filing cabinet. Tap a tab to open a different section without leaving the app.

Sheets slide up like a sticky note over your desk. Alerts are system pop-ups that demand a quick answer. Popovers are speech bubbles that point at a button.

navigation concepts and simple analogies
Concept What it does Analogy
TabView Switches between main app sections Labeled folders in a filing cabinet
tabItem Sets the icon and label for each tab The sticker on each folder tab
.sheet Presents a view that slides up from the bottom A sticky note placed over your current work
.alert Shows a system dialog with action buttons A yes-or-no question from the computer
.popover Shows a small floating panel near a control A speech bubble pointing at a button
dismiss Closes the current sheet or modal from inside it Peeling off the sticky note when you are done
isPresented binding Bool that controls whether a modal is visible A light switch that turns the modal on and off

Lab Scenario: Campus Hub App

You will build a three-tab campus app. The Home tab shows events. The Clubs tab lists clubs. The Profile tab shows student info. You will add a sheet to RSVP, an alert to confirm leaving a club, and a popover for help text.

Build three simple views. You will place them inside a TabView.

import SwiftUI

struct HomeTab: View {

Part 2: Build the TabView

Topics: Implementing tab bars with TabView and tabItem.

Wrap your three screens in a TabView. Each view gets a tabItem with a label and system image.

struct ContentView: View {
    var body: some View {
        TabView {
            HomeTab()
                .tabItem {
                    Label("Home", systemImage: "house")
                }

            ClubsTab()
                .tabItem {
                    Label("Clubs", systemImage: "person.3")
                }

            ProfileTab()
                .tabItem {
                    Label("Profile", systemImage: "person.crop.circle")
                }
        }
    }
}

Run the simulator and tap each tab. The tab bar stays at the bottom while the content changes.

Part 3: Present a Modal Sheet

Topics: Showing a modal sheet.

Add an RSVP button on the Home tab. Tapping it presents a sheet that slides up from the bottom.

struct HomeTab: View {
    @State private var showingRSVPSheet = false

    var body: some View {
        VStack(spacing: 16) {
            Text("Campus Events")
                .font(.title2)

            Button("RSVP to Study Night") {
                showingRSVPSheet = true
            }
            .buttonStyle(.borderedProminent)
        }
        .padding()
        .sheet(isPresented: $showingRSVPSheet) {
            RSVPSheet()
        }
    }
}

struct RSVPSheet: View {
    @Environment(\.dismiss) private var dismiss

    var body: some View {
        NavigationStack {
            VStack(spacing: 16) {
                Text("SwiftUI Study Night")
                    .font(.title2)
                Text("Friday at 6:00 PM in Room 204")
                    .foregroundColor(.secondary)
                Button("Confirm RSVP") {
                    dismiss()
                }
                .buttonStyle(.borderedProminent)
            }
            .padding()
            .navigationTitle("RSVP")
            .toolbar {
                ToolbarItem(placement: .cancellationAction) {
                    Button("Cancel") {
                        dismiss()
                    }
                }
            }
        }
    }
}

The $showingRSVPSheet binding is the light switch. Setting it to true opens the sheet. Calling dismiss() closes it.

Part 4: Show a Confirmation Alert

Topics: Showing alerts.

On the Clubs tab, add a button that asks the user to confirm before leaving a club.

struct ClubsTab: View {
    @State private var showingLeaveAlert = false

    var body: some View {
        VStack(spacing: 16) {
            Text("Campus Clubs")
                .font(.title2)

            Button("Leave Coding Club") {
                showingLeaveAlert = true
            }
            .buttonStyle(.bordered)
            .tint(.red)
        }
        .padding()
        .alert("Leave Club?", isPresented: $showingLeaveAlert) {
            Button("Cancel", role: .cancel) { }
            Button("Leave", role: .destructive) {
                // Club leave action would go here
            }
        } message: {
            Text("Are you sure you want to leave the Coding Club?")
        }
    }
}

Alerts are best for quick yes-or-no decisions. They block the screen until the user picks an action.

Part 5: Display a Help Popover

Topics: The popover.

On the Profile tab, add a help button that shows a small popover with extra info.

struct ProfileTab: View {
    @State private var showingHelpPopover = false

    var body: some View {
        VStack(spacing: 16) {
            Text("Your Profile")
                .font(.title2)
            Text("Name: Your Name")

            Button {
                showingHelpPopover = true
            } label: {
                Label("Help", systemImage: "questionmark.circle")
            }
            .popover(isPresented: $showingHelpPopover) {
                Text("Update your profile in Settings. Contact your instructor for login help.")
                    .padding()
                    .frame(width: 260)
            }
        }
        .padding()
    }
}

Popovers work well for short tips near the button that triggered them. On iPhone they often appear as a sheet-style panel.

Part 6: Optional Stretch — Presentation Detents

Topics: Presentation detents.

This section also introduces presentationDetents for controlling sheet height. Add this inside your RSVPSheet if you want to experiment:

.presentationDetents([.medium, .large])

This lets the user drag the sheet between half height and full height. Test it in the simulator on iOS 16 or later.

Troubleshooting

Tab Bar Does Not Appear

Make sure each child view has a .tabItem modifier directly attached. The root view must be a TabView, not a NavigationStack wrapping it incorrectly.

Sheet Does Not Open

Confirm you used .sheet(isPresented: $showingRSVPSheet) with the $ prefix. The Bool must be set to true inside the button action.

Dismiss Button Has No Effect

Add @Environment(\.dismiss) private var dismiss inside the sheet view. Call dismiss() as a function with parentheses.

Alert Shows No Message

Include the message: trailing closure in the .alert modifier. Both title and message are needed for clarity.
