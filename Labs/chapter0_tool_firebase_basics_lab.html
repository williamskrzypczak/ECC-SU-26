Chapter 0 Lab: Firebase Basics

Introduction

You will learn how to store app data in the cloud using Firebase. Firebase is Google's platform for mobile and web apps. Cloud Firestore is its online database.

Think of Firestore like a shared Google Sheet in the cloud. Your iOS app can read and write rows of data. A collection is a tab, a document is a row, and fields are the columns. You can complete this lab in about 15 minutes.

You will build CampusFirebase, a simple SwiftUI app that posts and lists campus announcements. Complete the Chapter 0 GitHub lab first so you can save your project.

Learning Objectives

After completing this lab, you will be able to:

  • Explain Firebase project, Console, Firestore, collection, document, and field
  • Create a Firebase project and register an iOS app
  • Create a Firestore database and add test data in the Console
  • Add the Firebase iOS SDK to an Xcode project with Swift Package Manager
  • Connect your app with GoogleService-Info.plist and FirebaseApp.configure()
  • Write and read documents from Firestore in SwiftUI

Key Ideas

Firestore is your cloud spreadsheet. Collections group related documents. Each document holds fields like text and dates. Your app talks to Firestore through the Firebase SDK.

GoogleService-Info.plist is your app's address book. It tells Firebase which project belongs to your iOS app. Without it, the SDK cannot find your database.

Firebase concepts and simple analogies
Concept What it does Analogy
Firebase project Container for your app's backend services A project folder in the cloud
Firebase Console Web dashboard to manage Firebase The control panel for your app's backend
Cloud Firestore Document database stored online A shared spreadsheet in the cloud
Collection Group of related documents A sheet tab
Document One record with fields A row in the sheet
Field A single value such as text or a date A cell
GoogleService-Info.plist Config file linking your iOS app to Firebase Your app's address book for the cloud
SDK Code library that talks to Firebase A translator between your app and Google's servers
Read / Write Load data from or save data to Firestore Opening or editing the shared sheet

Prerequisites

You need a Mac with Xcode 15 or later installed. You also need a Google account. Sign in at firebase.google.com if you do not have one yet.

Lab Instructions

Part 1: Create a Google Account and Open Firebase

Topics: Google sign-in and Firebase Console layout.

  1. Go to firebase.google.com.
  2. Click Get started or Go to console. Sign in with your Google account.
  3. Find these areas in the Console:
    • Project list — home screen showing your projects
    • Left sidebar — Build, Run, Analytics, and other sections
    • Project settings — gear icon for app registration and config files

Take a screenshot of the Firebase Console home screen. You will submit this later.

Part 2: Create a Firebase Project

Topics: Project creation and dashboard.

  1. Click Add project (or Create a project).
  2. Enter the project name: Campus Firebase Lab.
  3. Google Analytics is optional. You may disable it to keep setup simpler. Click Continue.
  4. Click Create project. Wait for setup to finish.
  5. Click Continue to open your project dashboard.

Part 3: Register Your iOS App

Topics: Bundle ID and config file download.

  1. On the project overview page, click the iOS icon to add an Apple app.
  2. Enter an iOS bundle ID. Example: com.yourname.CampusFirebase. Use your actual name in place of yourname.
  3. Enter the app nickname: CampusFirebase. Skip App Store ID for now.
  4. Click Register app.
  5. Download GoogleService-Info.plist. Save it where you can find it. You will add it to Xcode in Part 5.
  6. Click through the remaining setup steps. SDK installation comes next in Part 5.

Important: your Bundle ID in Firebase must exactly match the Bundle ID in Xcode later.

Part 4: Create a Firestore Database

Topics: Cloud Firestore, test mode, and manual data entry.

  1. In the left sidebar, open Build → Firestore Database.
  2. Click Create database.
  3. Choose Start in test mode. This allows read and write access for learning. Test mode is not safe for public apps. Click Next.
  4. Pick a Cloud Firestore location close to you. Click Enable.
  5. Click Start collection. Set the collection ID to announcements.
  6. Add a test document with these fields:
    • title (string): Welcome to MAD 157
    • author (string): Your Name
    • createdAt (timestamp): use today's date and time
  7. Click Save. Confirm the document appears in the Data tab.

Take a screenshot of the Firestore Data tab showing the announcements collection.

Part 5: Create Xcode Project and Add Firebase SDK

Topics: Xcode project, config file, and Swift Package Manager.

  1. Open Xcode. Choose File → New → Project.
  2. Select iOS → App. Click Next.
  3. Set the product name to CampusFirebase. Set Interface to SwiftUI and Language to Swift.
  4. Set the Bundle Identifier to match Part 3 exactly (for example, com.yourname.CampusFirebase).
  5. Drag GoogleService-Info.plist into the Xcode project navigator. Check Copy items if needed and your app target.
  6. Choose File → Add Package Dependencies.
  7. Enter the URL: https://github.com/firebase/firebase-ios-sdk. Click Add Package.
  8. Select FirebaseFirestore. Add it to the CampusFirebase target. FirebaseCore is included automatically.

Part 6: Write SwiftUI Read and Write Code

Topics: Firebase initialization and Firestore read/write.

Step 6.1: Initialize Firebase in your app entry file

Open CampusFirebaseApp.swift (or your app entry file). Add Firebase setup:

import SwiftUI
import FirebaseCore

@main
struct CampusFirebaseApp: App {
    init() {
        FirebaseApp.configure()
    }

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}

Step 6.2: Build ContentView with read and write

Replace ContentView.swift with the code below. Replace Your Name with your actual name.

import SwiftUI
import FirebaseFirestore

struct Announcement: Identifiable {
    let id: String
    let title: String
    let author: String
}

struct ContentView: View {
    @State private var titleText = ""
    @State private var announcements: [Announcement] = []
    @State private var statusMessage = "Ready"

    private let db = Firestore.firestore()

    var body: some View {
        NavigationStack {
            VStack(spacing: 16) {
                TextField("Announcement title", text: $titleText)
                    .textFieldStyle(.roundedBorder)

                Button("Post") {
                    postAnnouncement()
                }
                .buttonStyle(.borderedProminent)

                Text(statusMessage)
                    .font(.footnote)
                    .foregroundColor(.secondary)

                List(announcements) { item in
                    VStack(alignment: .leading, spacing: 4) {
                        Text(item.title)
                            .font(.headline)
                        Text("By \(item.author)")
                            .font(.subheadline)
                            .foregroundColor(.secondary)
                    }
                }
            }
            .padding()
            .navigationTitle("Campus Announcements")
            .onAppear {
                loadAnnouncements()
            }
        }
    }

    private func postAnnouncement() {
        let trimmed = titleText.trimmingCharacters(in: .whitespacesAndNewlines)
        guard !trimmed.isEmpty else {
            statusMessage = "Enter a title first."
            return
        }

        db.collection("announcements").addDocument(data: [
            "title": trimmed,
            "author": "Your Name",
            "createdAt": Timestamp(date: Date())
        ]) { error in
            if let error = error {
                statusMessage = "Write failed: \(error.localizedDescription)"
                return
            }
            titleText = ""
            statusMessage = "Posted to Firestore."
            loadAnnouncements()
        }
    }

    private func loadAnnouncements() {
        db.collection("announcements")
            .order(by: "createdAt", descending: true)
            .getDocuments { snapshot, error in
                if let error = error {
                    statusMessage = "Read failed: \(error.localizedDescription)"
                    return
                }

                announcements = snapshot?.documents.compactMap { doc in
                    let data = doc.data()
                    guard let title = data["title"] as? String,
                          let author = data["author"] as? String else {
                        return nil
                    }
                    return Announcement(id: doc.documentID, title: title, author: author)
                } ?? []
            }
    }
}

Troubleshooting

Bundle ID Mismatch

The Bundle ID in Firebase must exactly match your Xcode target. Check both under Project Settings → General and in the Firebase Console app registration.

GoogleService-Info.plist Not Found

Click the file in Xcode. Open the File inspector on the right. Confirm your app target is checked under Target Membership.

Permission Denied on Write

Confirm Firestore is in test mode. Open Firestore → Rules in the Console. Test mode allows open read/write for a limited time.

Package Resolve Fails

Check your internet connection. Use the exact URL https://github.com/firebase/firebase-ios-sdk. Try File → Packages → Reset Package Caches.

Empty List in the App

Check that the collection name is exactly announcements. Confirm documents exist in the Firestore Data tab. Read the status message at the bottom of the app for error details.
