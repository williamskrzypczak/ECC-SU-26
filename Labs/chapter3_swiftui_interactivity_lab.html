Chapter 3 Lab: Adding Interactivity to a SwiftUI View

Introduction

You will turn a static event card into an interactive RSVP screen that responds to taps, toggles, and text input.

Until now, your views were like posters on a wall. They looked nice but could not react. This chapter teaches you how to make the screen respond when users tap, type, or choose options.

You can complete this lab in about 15 minutes.

Learning Objectives

After completing this lab, you will be able to:

  • Pass values into a view when you create it
  • Use @State to store data that changes inside a view
  • Respond to button taps and update the UI dynamically
  • Bind controls like Toggle, Picker, and TextField with $
  • Share state with a child view using @Binding
  • Explain the difference between @State and @Binding

Key Ideas

Views are structs. They are normally immutable, like a printed form. @State gives the view a whiteboard it can erase and rewrite when the user acts.

Bindings are two-way streets. The parent owns the data. The child gets a @Binding so both views stay in sync, like sharing one notebook instead of copying pages.

interactivity concepts and simple analogies
Concept What it does Analogy
Passing values at creation Send data into a view through its initializer Writing a name tag before handing it to someone
@State Stores changeable data owned by this view A whiteboard only this view can update
Button action Runs code when the user taps A doorbell that triggers something inside
$ binding prefix Creates a two-way link to a @State value A shared notebook both views can write in
@Binding Lets a child view read and write parent state A remote control for someone else's whiteboard
Toggle On/off switch bound to a Bool value A light switch on the wall
Picker Dropdown or wheel to choose one option A cafeteria menu with set choices
TextField Text input that updates as the user types A fill-in-the-blank line on a form

Lab Scenario: Campus Event RSVP

You will upgrade your campus event card into an RSVP form. Students can tap a button to sign up, toggle reminders, pick a session, and type their name. The screen updates live as they interact.

Lab Instructions

Part 1: Pass Values at View Creation

Topics: Passing values at view creation.

This lab shows that you can pass data into a view when you build it. Create an EventCard that accepts an event title and location as parameters.

import SwiftUI

struct EventCard: View {
    let eventTitle: String
    let location: String

    var body: some View {
        VStack(alignment: .leading, spacing: 8) {
            Text(eventTitle)
                .font(.title2)
                .fontWeight(.bold)
            Text(location)
                .font(.subheadline)
                .foregroundColor(.secondary)
        }
        .frame(maxWidth: .infinity, alignment: .leading)
        .padding()
        .background(Color(.systemGray6))
        .cornerRadius(12)
    }
}

Preview it with different values to see how passing data at creation works:

#Preview {
    EventCard(eventTitle: "SwiftUI Study Night", location: "Room 204")
}

Part 2: Add @State and a Button

Topics: Responding to a button tap and Property wrappers.

Static views cannot change. Add @State to track how many students tapped RSVP. Use a Button to increase the count.

struct ContentView: View {
    @State private var rsvpCount = 0

    var body: some View {
        VStack(spacing: 20) {
            EventCard(
                eventTitle: "SwiftUI Study Night",
                location: "MCC Student Center, Room 204"
            )

            Text("RSVPs so far: \(rsvpCount)")
                .font(.headline)

            Button("Count Me In") {
                rsvpCount += 1
            }
            .buttonStyle(.borderedProminent)
        }
        .padding()
    }
}

Tap the button in the preview or simulator. The count should go up each time. SwiftUI redraws the view when @State changes.

Part 3: Change Appearance Dynamically

Topics: Changing a view appearance dynamically using modifiers.

Add a Bool state value to track whether the current user has RSVP'd. Change the screen when the value is true.

@State private var isGoing = false

// Inside the VStack, below the button:
if isGoing {
    Text("You are signed up!")
        .foregroundColor(.green)
        .fontWeight(.semibold)
} else {
    Text("Tap the button to RSVP.")
        .foregroundColor(.secondary)
}

Button(isGoing ? "Cancel RSVP" : "Count Me In") {
    isGoing.toggle()
    if isGoing {
        rsvpCount += 1
    } else if rsvpCount > 0 {
        rsvpCount -= 1
    }
}
.buttonStyle(.borderedProminent)
.tint(isGoing ? .red : .blue)

Notice how the button label, color, and message all react to isGoing. This is a responsive UI.

Part 4: Bidirectional Binding with Toggle and TextField

Topics: Bidirectional binding with Picker and TextField examples.

Add a reminder toggle and a name field. Use the $ prefix to create two-way bindings.

@State private var wantsReminder = false
@State private var studentName = ""

// Add inside the VStack:
Divider()

Toggle("Send me a reminder", isOn: $wantsReminder)

TextField("Enter your name", text: $studentName)
    .textFieldStyle(.roundedBorder)

if !studentName.isEmpty {
    Text("Hello, \(studentName)!")
        .font(.callout)
        .foregroundColor(.blue)
}

Type in the TextField and flip the Toggle. Both controls stay synced with your @State variables instantly.

Part 5: Add a Picker for Session Choice

Topics: Bidirectional binding with Picker.

Let students pick a session time. Store the choice in @State and bind it to a Picker.

@State private var selectedSession = "6:00 PM"

let sessions = ["5:00 PM", "6:00 PM", "7:00 PM"]

// Add inside the VStack:
Picker("Choose a session", selection: $selectedSession) {
    ForEach(sessions, id: \.self) { session in
        Text(session).tag(session)
    }
}
.pickerStyle(.segmented)

Text("You chose the \(selectedSession) session.")
    .font(.caption)
    .foregroundColor(.secondary)

Part 6: Share State with a Child View Using @Binding

Topics: Final notes on Subviews and @Binding.

Extract the RSVP button into its own view. Pass a @Binding so the child can change the parent's state. This is the two-way street from the analogy box.

struct RSVPButton: View {
    @Binding var isGoing: Bool
    @Binding var rsvpCount: Int

    var body: some View {
        Button(isGoing ? "Cancel RSVP" : "Count Me In") {
            isGoing.toggle()
            if isGoing {
                rsvpCount += 1
            } else if rsvpCount > 0 {
                rsvpCount -= 1
            }
        }
        .buttonStyle(.borderedProminent)
        .tint(isGoing ? .red : .blue)
    }
}

In ContentView, replace your inline button with the child view. Pass bindings using $:

RSVPButton(isGoing: $isGoing, rsvpCount: $rsvpCount)

Troubleshooting

Button Tap Does Not Update the Screen

Make sure the changing value uses @State. A regular var will not trigger a redraw. Mark state properties as private in the owning view.

Toggle or TextField Does Not Change Data

Check that you used the $ prefix in the control. Toggle("Label", isOn: $wantsReminder) is correct. Without $, the control cannot write back to state.

Child View Cannot Modify Parent Value

The child needs @Binding, not @State. The parent passes $propertyName. Both views must reference the same source of truth.

Picker Shows Empty Selection

Make sure .tag() values on each Text inside the Picker match the type stored in @State. Your default selectedSession must match one of the tags.
