Chapter 0 Lab: Figma Basics

Introduction

You will learn how to design app screens in Figma before writing code. Figma is a free design tool that runs in your browser. You sketch layouts, test click flows, and share your work with others.

Think of Figma like a floor plan before building a house. You plan the layout first, then build it in code. You can complete this lab in about 15 minutes. Complete it after the GitHub lab and before the Chapter 1 SwiftUI lab.

You will design the same Campus Welcome screen you build in Chapter 1. All work happens at figma.com. No plugins or desktop app are required.

Learning Objectives

After completing this lab, you will be able to:

  • Navigate the Figma interface (canvas, layers, properties panel, toolbar)
  • Create frames, shapes, and text
  • Apply colors and basic typography
  • Use auto layout to stack elements (similar to SwiftUI VStack)
  • Build a reusable component with instances
  • Connect two frames with a simple prototype
  • Share a file link and export a PNG

Key Ideas

Figma is your sketchbook. You draw screens before coding them. Frames represent phone screens. Layers are the individual pieces inside each screen.

Auto layout stacks things for you. It works like a SwiftUI VStack. Components let you reuse a button design everywhere without redrawing it.

Figma concepts and SwiftUI parallels
Concept What it does SwiftUI parallel
Canvas The workspace where you design Xcode canvas or preview
Frame A sized artboard, such as an iPhone screen Device screen bounds
Layer Any object listed in the left panel A view in a hierarchy
Auto layout Stacks and spaces objects automatically VStack, HStack, spacing
Component A reusable master design Reusable view or button style
Instance A copy linked to a component Same button used in many places
Prototype Clickable preview linking screens Navigation between views
Export Save a frame as an image file Screenshot of your app

Prerequisites

You need a computer with a modern web browser and internet access. Create a free Figma account at figma.com/signup if you do not have one yet. Complete the Chapter 0 GitHub lab first.

Lab Instructions

Part 1: Set Up Figma and Tour the Interface

Topics: Figma account, design file, and main panels.

  1. Go to figma.com and sign in.
  2. Click New design file in the file browser.
  3. Rename the file to Campus Welcome Wireframe (click the file name at the top).
  4. Find these four areas on screen:
    • Toolbar — top of the window (selection, frame, shape, text tools)
    • Layers panel — left sidebar (lists every object)
    • Canvas — center area where you design
    • Properties panel — right sidebar (size, color, typography)

Take a screenshot of your empty file. Label or note the four areas listed above. You will submit this later.

Part 2: Create an iPhone Frame

Topics: Frames and device presets.

  1. Press F on your keyboard, or click the Frame tool in the toolbar.
  2. In the right panel, open the device preset list. Choose iPhone 15 (or a similar iPhone size).
  3. Click on the canvas to place the frame.
  4. In the layers panel, double-click the frame name. Rename it to Welcome Screen.

A frame is like a phone screen boundary. Everything for one screen lives inside its frame.

Part 3: Add Shapes and Text

Topics: Text layers, rectangles, colors, and typography.

  1. Press T to select the Text tool. Click inside the frame and type Campus Welcome.
  2. In the properties panel, set the font size to 28 and weight to Bold. Set the color to blue (for example, #005FCC).
  3. Add a second text layer: Built by Your Name. Set size to 18 and color to gray (for example, #666666).
  4. Add a third text layer: Learning SwiftUI one view at a time. Set size to 14 and color to light gray.
  5. Press R to draw a rectangle below the text. Set the corner radius to 8 in the properties panel.
  6. Press T again. Click on the rectangle and type Get Started. Center the label and set the text color to white.
  7. Set the rectangle fill to blue (matching the title color).

Replace Your Name with your actual name. Arrange the elements roughly in a vertical stack for now.

Part 4: Use Auto Layout

Topics: Auto layout, spacing, and padding.

Auto layout stacks elements automatically. It works like a SwiftUI VStack with spacing and padding.

  1. Select all four elements inside the frame (title, name, tagline, and button group). Shift-click each one.
  2. Press Shift+A or click the + next to Auto layout in the right panel.
  3. Set direction to Vertical.
  4. Set item spacing to 12 or 16 pixels.
  5. Set padding to 24 pixels on all sides.
  6. Select the auto layout frame. Set alignment to Center (horizontal and vertical).
  7. Resize or reposition the auto layout group so it sits centered in the Welcome Screen frame.

Take a screenshot showing the auto layout settings in the properties panel.

Part 5: Create a Component

Topics: Components, instances, and reusable design.

A component is a master design. When you change the master, every copy updates too.

  1. Select the button rectangle and its Get Started label together.
  2. Right-click and choose Create component, or press Cmd+Option+K.
  3. In the layers panel, rename the component to PrimaryButton.
  4. Double-click the button label on the canvas. Change the text to Get Started (confirm it still reads correctly).
  5. Notice the purple diamond icon in the layers panel. That marks a component.

Take a screenshot showing PrimaryButton in the layers panel with the component icon.

Part 6: Prototype a Second Screen

Topics: Duplicating frames, prototype connections, and Present mode.

  1. Select the Welcome Screen frame in the layers panel.
  2. Hold Option and drag the frame to the right to duplicate it.
  3. Rename the copy to Detail Screen.
  4. On the Detail Screen, change the title text to Welcome aboard!
  5. Click the Prototype tab in the right panel (next to Design).
  6. On the Welcome Screen, click the Get Started button. Drag the blue circle to the Detail Screen frame.
  7. Set the interaction to On tap → Navigate to → Detail Screen.
  8. Click the Present button (play icon) in the top right. Click Get Started to test the flow.

Take a screenshot of the blue prototype arrow connecting the two frames. Take a second screenshot of Present mode showing the Detail Screen.

Part 7: Share and Export

Topics: Sharing files and exporting images.

  1. Click the Share button in the top right.
  2. Set link access to Anyone with the link can view.
  3. Copy the share URL.
  4. Return to the Design tab. Select the Welcome Screen frame (not an inner layer).
  5. In the properties panel, scroll to Export. Click +. Choose PNG at 1x.
  6. Click Export Welcome Screen. Save the file as campus-welcome-wireframe.png.

Troubleshooting

Cannot Find the Frame Tool

Press F on your keyboard. Or click and hold the shape tool in the toolbar to find Frame.

Auto Layout Looks Wrong

Check that direction is set to Vertical in the properties panel. Make sure you selected all items before pressing Shift+A.

Prototype Does Not Play

Confirm you see a blue arrow connecting the button to the second frame. Click Present (play icon), not just the canvas preview.

Export Is Blank or Cropped

Select the outer Welcome Screen frame in the layers panel. Do not select an inner text layer or auto layout group.

Component Icon Not Showing

Select both the rectangle and the text label together before choosing Create component. The purple diamond appears in the layers panel.
