Chapter 0 Lab: Slack Basics

Introduction

You will learn how teams communicate using Slack. Slack replaces long email chains with organized channels where everyone can chat, share files, and ask questions.

Think of Slack like a group chat for a whole class or dev team. Each channel covers one topic. This lab takes about 60 to 75 minutes. Complete it after the GitHub, Figma, and Trello labs and before the Chapter 1 SwiftUI lab.

You will set up a practice workspace called MAD 157 Team Hub. Use the Slack web app at slack.com or the desktop app.

Learning Objectives

After completing this lab, you will be able to:

  • Explain workspace, channel, direct message, thread, mention, and reaction
  • Join or create a workspace and navigate the Slack interface
  • Create channels and send formatted messages
  • Reply in a thread to keep conversations organized
  • Use @mentions and emoji reactions
  • Share a file or code snippet
  • Search messages and copy a channel link to share

Key Ideas

A workspace is your team's home base. Channels are rooms for different topics. Direct messages are private chats between two people.

Threads keep conversations tidy. Instead of flooding a channel, you reply in a thread tied to one message. Reactions give quick feedback without a full reply.

Slack concepts and real-world parallels
Concept What it does Real-world parallel
Workspace The home for one team or class A shared office building
Channel A topic-based chat room A classroom or meeting room
Direct message (DM) Private chat between people A private text conversation
Message A post in a channel or DM A comment in a group chat
Thread Replies tied to one message Comment thread on a single post
@mention Tag a person to notify them Saying someone's name in a meeting
Reaction Emoji reply on a message A quick thumbs-up
Pin Highlight an important message Sticky note on the bulletin board
Search Find past messages Ctrl+F for the whole team chat

Prerequisites

You need a computer with a modern web browser and internet access. Create a free Slack account at slack.com/get-started or join an instructor-provided workspace invite link. Complete the Chapter 0 GitHub, Figma, and Trello labs first.

Lab Instructions

Part 1: Set Up Slack and Tour the Interface

Topics: Slack account, workspace, and main panels.

  1. Go to slack.com/get-started and sign in or create an account.
  2. Create a new workspace named MAD 157 Team Hub. Or open an invite link from your instructor if one is provided.
  3. Find these areas on screen:
    • Sidebar — left side (channels and direct messages)
    • Message list — center area showing channel messages
    • Compose box — bottom of the screen where you type
    • Details panel — right side (channel info, members, pins)

Take a screenshot of your workspace. Note the four areas listed above. You will submit this later.

Part 2: Create Channels

Topics: Public channels and channel topics.

  1. Click the + next to Channels in the sidebar.
  2. Create three public channels: general, help, and campus-welcome.
  3. Open #campus-welcome. Click the channel name at the top to open details.
  4. Set the channel topic to: Discuss the Chapter 1 Campus Welcome SwiftUI app.
  5. Click each channel in the sidebar to confirm you can switch between them.

Take a screenshot of the sidebar showing all three channels.

Part 3: Send Messages

Topics: Posting messages and basic formatting.

  1. Open #general. Post an introduction with your name, the course (MAD 157), and one sentence about your coding experience.
  2. Open #campus-welcome. Post a message about your Figma wireframe or Trello board from the earlier labs.
  3. Use basic formatting in your #campus-welcome message:
    • Wrap one word in asterisks for bold: *bold*
    • Wrap a project name in backticks for inline code: `CampusWelcome`

Example message: Finished my *Figma* wireframe for the `CampusWelcome` app. Ready to start coding in Chapter 1!

Take a screenshot of your formatted message in #campus-welcome.

Part 4: Use Threads

Topics: Organized replies without cluttering the channel.

  1. Go to #general and find your introduction message.
  2. Hover over the message. Click Reply in thread (speech bubble icon).
  3. Type a follow-up, such as: Looking forward to building my first SwiftUI app!
  4. Send the reply. Notice the thread opens in a side panel without adding noise to the main channel.

Take a screenshot of the open thread with at least one reply.

Part 5: Mentions and Reactions

Topics: Notifying people and quick emoji feedback.

  1. Open #help. Post a question about the course or a lab step.
  2. Type @ followed by your name to @mention yourself (or a classmate if on a shared workspace). Select the name from the popup list.
  3. Find any message in your workspace. Hover over it and click the Add reaction icon (smiley face).
  4. Add a reaction such as a thumbs-up or checkmark.

In your notes, write one sentence explaining why threads and reactions reduce noise in a busy channel.

Part 6: Share a File or Snippet

Topics: Uploading files and sharing code.

  1. Open #campus-welcome.
  2. Click the + icon to the left of the compose box.
  3. Choose one option:
    • Upload a screenshot from your Figma or Trello lab, or
    • Create a Snippet and paste a few lines from your GitHub lab ContentView.swift
  4. Add a caption explaining what you shared. Example: My Figma wireframe for the welcome screen.
  5. Send the message.

Take a screenshot showing your message with the uploaded file or snippet and an emoji reaction.

Part 7: Search and Share a Link

Topics: Finding messages and sharing channel access.

  1. Click the Search bar at the top of Slack.
  2. Search for a keyword from one of your earlier messages, such as CampusWelcome or Figma.
  3. Confirm your message appears in the search results.
  4. Open #campus-welcome channel details. Copy the channel link.
  5. Optional stretch: hover your file-sharing message and click Pin to channel so it stays easy to find.

Reflection Questions

Answer each question in one sentence:

  1. What is the difference between a channel and a direct message?
  2. Why reply in a thread instead of the main channel?
  3. When would you use an @mention?

Troubleshooting

Cannot Create a Workspace

Use an instructor invite link if one is provided. Or sign in with a different email address and try again.

Channel Not Visible in Sidebar

Click the + next to Channels. Choose Create a channel or browse channels to join.

Thread Panel Missing

Hover over a message until the action icons appear. Click Reply in thread (speech bubble icon).

File Upload Fails

Check that your file is under 5 MB. PNG screenshots from Figma or Trello work well.

@mention Not Working

Type @ and wait for the name popup. Click a name from the list instead of typing the full name manually.
