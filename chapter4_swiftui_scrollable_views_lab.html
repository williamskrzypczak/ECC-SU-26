Chapter 4 Lab: Scrollable Views and Lists

Introduction

This lab reinforces Chapter 4 of An iOS Developer's Guide to SwiftUI by Michele Fadda. You will build a campus club directory that scrolls through many items, shows a list, controls the keyboard, and jumps to sections on demand.

Real apps often have more content than fits on one screen. This chapter teaches you how to display rows of data and let users scroll through them smoothly.

Learning Objectives

After completing this lab, you will be able to:

  • Use ForEach to iterate over data and build repeated views
  • Display long content inside a ScrollView
  • Build a native iOS List from the same data
  • Control the keyboard with @FocusState in a form
  • Scroll to a specific item using ScrollViewReader and .id()
  • Add a navigation title to your screen

Key Ideas from Chapter 4

ForEach is like a stamp machine. You define one row design, and SwiftUI stamps it out for every item in your data.

ScrollView is like unrolling a long poster. List is like a lined notebook with built-in row styling. ScrollViewReader is an elevator button that jumps to a specific floor.

Chapter 4 scrollable view concepts and simple analogies
Concept What it does Analogy

ScrollView Lets users swipe through content taller than the screen Unrolling a long poster vertically

Lab Scenario: Campus Club Directory

You will build a scrollable directory of campus clubs. Students can browse clubs, search by name, jump to a letter section, and view clubs in both a scroll view and a list.

Lab Instructions

Part 1: Create Sample Data and a Row View

Covers Chapter 4: Iterating views.

Start with a simple data model and a reusable row. Each club has a name and a category letter for grouping.

import SwiftUI

struct Club: Identifiable {
    let id = UUID()
    let name: String
    let section: String
}

let campusClubs: [Club] = [
    Club(name: "Coding Club", section: "C"),
    Club(name: "Creative Writing", section: "C"),
    Club(name: "Debate Team", section: "D"),
    Club(name: "Design Society", section: "D"),
    Club(name: "Esports League", section: "E"),
    Club(name: "Engineering Club", section: "E"),
    Club(name: "Film Club", section: "F"),
    Club(name: "Fitness Group", section: "F"),
    Club(name: "Gaming Guild", section: "G"),
    Club(name: "Garden Club", section: "G")
]

struct ClubRow: View {
    let club: Club

    var body: some View {
        HStack {
            Text(club.section)
                .font(.headline)
                .foregroundColor(.white)
                .frame(width: 32, height: 32)
                .background(Color.blue)
                .cornerRadius(8)
            Text(club.name)
                .font(.body)
            Spacer()
        }
        .padding(.vertical, 4)
    }
}

Part 2: Iterate with ForEach Inside a ScrollView

Covers Chapter 4: Scroll views and iterating views.

Wrap your clubs in a ScrollView and a LazyVStack. Use ForEach to build one row per club.

struct ContentView: View {
    var body: some View {
        NavigationStack {
            ScrollView {
                LazyVStack(alignment: .leading, spacing: 12) {
                    ForEach(campusClubs) { club in
                        ClubRow(club: club)
                    }
                }
                .padding()
            }
            .navigationTitle("Campus Clubs")
        }
    }
}

Run the preview or simulator. Swipe up and down to scroll through all clubs. LazyVStack only builds rows as they appear, which helps performance.

Part 3: Display the Same Data in a List

Covers Chapter 4: Lists.

SwiftUI also provides a List for scrollable rows with built-in styling. Add a second tab or replace the ScrollView temporarily to compare.

List {
    ForEach(campusClubs) { club in
        ClubRow(club: club)
    }
}
.navigationTitle("Club List")

Part 4: Control the Keyboard with @FocusState

Covers Chapter 4: Hiding or showing the keyboard in a form using @FocusState.

Add a search field above the club list. Use @FocusState to control which field has the keyboard spotlight.

struct ContentView: View {
    @State private var searchText = ""
    @FocusState private var isSearchFocused: Bool

    var filteredClubs: [Club] {
        if searchText.isEmpty {
            return campusClubs
        }
        return campusClubs.filter {
            $0.name.localizedCaseInsensitiveContains(searchText)
        }
    }

    var body: some View {
        NavigationStack {
            VStack(spacing: 12) {
                HStack {
                    TextField("Search clubs", text: $searchText)
                        .textFieldStyle(.roundedBorder)
                        .focused($isSearchFocused)

                    Button("Done") {
                        isSearchFocused = false
                    }
                }
                .padding(.horizontal)

                List {
                    ForEach(filteredClubs) { club in
                        ClubRow(club: club)
                    }
                }
            }
            .navigationTitle("Campus Clubs")
        }
    }
}

Tap the search field to show the keyboard. Tap Done to dismiss it. Setting isSearchFocused = false hides the keyboard.

Part 5: Jump to a Section with ScrollViewReader

Covers Chapter 4: Controlling scrolling programmatically using ScrollViewReader.

Add section headers and jump buttons. Tag each section with .id() so ScrollViewReader can scroll to it.

let sections = ["C", "D", "E", "F", "G"]

// Replace the List/ScrollView body with this scrollable layout:
ScrollViewReader { proxy in
    VStack(spacing: 8) {
        HStack {
            ForEach(sections, id: \.self) { section in
                Button(section) {
                    withAnimation {
                        proxy.scrollTo(section, anchor: .top)
                    }
                }
                .buttonStyle(.bordered)
            }
        }
        .padding(.horizontal)

        ScrollView {
            LazyVStack(alignment: .leading, spacing: 16) {
                ForEach(sections, id: \.self) { section in
                    Text(section)
                        .font(.title2)
                        .fontWeight(.bold)
                        .id(section)

                    ForEach(campusClubs.filter { $0.section == section }) { club in
                        ClubRow(club: club)
                    }
                }
            }
            .padding()
        }
    }
}

Tap a letter button. The scroll view should jump to that section. The .id(section) modifier is the room number that ScrollViewReader looks for.

Troubleshooting

ForEach Shows Duplicate ID Error

Every item needs a unique identifier. Use Identifiable with a UUID() id, or pass id: \.name if names are guaranteed unique.

ScrollViewReader Does Not Scroll

Confirm the target view has a matching .id() value. The id passed to scrollTo() must exactly match. Wrap the scroll call in withAnimation for a smoother jump.

Keyboard Will Not Dismiss

Check that the TextField uses .focused($isSearchFocused) and the Done button sets isSearchFocused = false. Import SwiftUI is required; no extra frameworks needed.

List and ScrollView Look Identical

List adds default row separators and inset styling. ScrollView gives you full layout control. Try both side by side in previews to see the difference.
